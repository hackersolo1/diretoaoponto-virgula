Variáveis (var, let, const)

8 minutos
12.5k visualizações
15 Dez 2024

Neste episódio você vai entender de uma vez por todas as diferenças entre var, let e const no JavaScript. Vamos ver quando usar cada uma, os problemas do var, e por que let e const são a escolha moderna. Direto ao ponto, sem enrolação!

Exemplo 1: var vs let vs const 
// VAR - escopo de função, pode ser redeclarada
var nome = "João";
var nome = "Maria"; // ✅ Funciona (mas não deveria!)

// LET - escopo de bloco, não pode ser redeclarada
let idade = 25;
let idade = 30; // ❌ Erro!
idade = 30; // ✅ Reatribuição funciona

// CONST - escopo de bloco, não pode ser reatribuída
const PI = 3.14159;
PI = 3.14; // ❌ Erro!
Exemplo 2: Escopo de bloco 
if (true) {
  var x = 10;
  let y = 20;
  const z = 30;
}

console.log(x); // ✅ 10 (var vaza do bloco)
console.log(y); // ❌ Erro (let tem escopo de bloco)
console.log(z); // ❌ Erro (const tem escopo de bloco)
Exemplo 3: Quando usar cada uma 
// Use CONST por padrão (não vai mudar)
const API_URL = "https://api.exemplo.com";
const usuarios = ["Ana", "Bruno", "Carlos"];

// Use LET quando o valor vai mudar
let contador = 0;
for (let i = 0; i < 10; i++) {
  contador++;
}

// EVITE VAR (código legado apenas)
var antigoJeito = "não use mais";
  • var tem escopo de função e pode causar bugs inesperados. Evite usar em código novo.
  • let tem escopo de bloco e permite reatribuição. Use quando o valor vai mudar.
  • const tem escopo de bloco e não permite reatribuição. Use como padrão para valores que não mudam.
  • Regra de ouro: Sempre use const por padrão. Só mude para let se realmente precisar reatribuir.
  • Objetos e arrays: Com const, você pode modificar o conteúdo, mas não pode reatribuir a variável.
  • Hoisting: var é "elevada" (hoisted) para o topo do escopo, let e const não. Isso evita bugs.
